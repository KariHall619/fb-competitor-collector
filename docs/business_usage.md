# FB 竞品采集 Skill 使用方式

这套工具给业务同学在 Codex 里用自然语言操作，不需要记脚本命令。

## 使用前准备

1. 用平时工作的 Chrome 登录 Facebook。
2. 在同一个 Chrome profile 里启用 Codex Chrome Extension。
3. 打开需要采集的 Facebook 主页，并确认肉眼能看到帖子列表。
4. 飞书 CLI 需要保持用户身份登录。

## 常用说法

检查工具状态：

```text
使用 FB 竞品采集 skill，检查一下现在能不能用。
```

检查会确认飞书登录、输出表配置、用户身份限制、Codex Chrome Extension 是否可用。

测试当前页面是否能读到正文：

```text
使用 FB 竞品采集 skill，试一下当前 Chrome 页面能不能抓到帖子正文，不要写飞书。
```

采集当前打开页面：

```text
使用 FB 竞品采集 skill，采集当前 Chrome 页面里可见的帖子，先保存到内容库。
```

采集竞品调研账户第一个账号：

```text
使用 FB 竞品采集 skill，采集竞品调研账户第一个账号今天的帖子，并同步到飞书。
```

导入已有抓取结果：

```text
使用 FB 竞品采集 skill，把这份抓取结果导入内容库。
文件路径是：<这里填文件路径>
```

筛选某一天竞品帖子：

```text
使用 FB 竞品采集 skill，筛选 5 月 21 日的竞品帖子，并写入飞书筛选结果。
```

筛选高浏览量帖子：

```text
使用 FB 竞品采集 skill，筛选浏览量大于 10 万的帖子。
```

筛选高点赞文本帖：

```text
使用 FB 竞品采集 skill，筛选点赞量超过 100 的文本帖。
```

## 业务侧需要提供什么

当前页面采集时，需要业务同学先打开目标 Facebook 页面。

手动导入时，至少需要提供以下三列：

```text
帖子链接
文章链接
简述
```

如果有这些字段，也可以一起提供：

```text
账号名
账号主页链接
账号类型：竞品账号 / 内部主页账号
发帖时间
帖子类型
浏览量
点赞量
```

## 重要说明

- 只有 Codex Chrome Extension 路线用于实时采集。
- 如果插件不可用，系统会直接停止并提示修复插件或 Chrome profile 设置。
- Facebook 页面必须是业务同学肉眼能看到帖子列表的正常登录页面。
- 如果页面出现登录入口、游客预览、只显示一条预览帖子、空白动态壳、评论片段或没有真实帖子，系统会立即停止并提示 `human_intervention_required`，需要人工登录/确认页面后再重试。
- 重复的帖子链接不会重复入库。
- 没有浏览量/点赞量时不会阻塞入库，会标记为“互动数据未确认”；但如果业务要求按互动数据筛选，缺失互动数据的记录不会命中。
- 发帖时间要求至少到小时，格式为 `2026年5月19日 17:00`。
- `19min`、`1h`、`16h`、`1d` 这类相对时间会先尝试通过 Facebook 精确时间悬停提示或页面 DOM 属性补齐；如果补不到精确时间，则按采集时间估算，并在表格发帖时间中标注 `约`。
- 时间悬停由 Skill 在专用采集 Chrome 中自动完成：优先使用不会移动真实鼠标的页面内自动 hover；必要时再使用 CDP 自动移动鼠标到时间 UI。业务同学不需要手动悬停时间。
- 只有登录失效、游客预览、验证码/风控、专用采集 profile 不对、页面没有真实加载出帖子这类阻断问题，才需要人工介入。
- `reel/photo/watch/video/post/story` 都会先作为 FB 内容候选保留，不会因为缺父帖链接被丢弃。
- 父帖链接只作为优先去重依据；抓不到父帖时保留原始 FB 内容链接，后续再处理重复。
- 正式入表必须补齐评论区或评论回复里的引流链接，并确认最终落到外部网站。
- 故事概要必须来自引流落地页/文章内容的中文总结，不能直接复制 Facebook 帖子正文。
- 较短帖子不会因为正文短而丢弃；只要 FB 内容链接有效，会进入待补全状态，继续补引流落地链接、文章概要和发帖时间。完全没有时间信号的记录不写正式表。
- 同步飞书前，需要确保 `FB竞品帖子链接` 输出表格 URL 已配置。
- 账号来源表只用于读取账号清单，不写入采集结果。
- 第一阶段只做采集、入库、筛选、同步飞书；不做生文、发布和引流帖生成。

## 具体日期采集说明

如果业务说“抓 5 月 27 日”，系统不能只按“过去 24 小时”处理。正确逻辑是：

1. 先抓目标账号附近时间段的可见帖子。
2. 能看到绝对时间的，按绝对时间写入 `posted_at`。
3. 只能看到 `19min/1h/16h/1d` 等相对时间且没有精确时间的，按采集时间估算 `posted_at`。
4. 写入表格时在发帖时间前标注 `约`，明确这是大概时间。
